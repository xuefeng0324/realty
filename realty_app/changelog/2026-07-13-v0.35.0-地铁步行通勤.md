# v0.35.0 地铁步行通勤

> 2026-07-13

## 🎯 这次改了什么

新增 **"🚶 地铁步行通勤 Top"** 卡片 — 展示每个小区到 **最近地铁站** 的步行时长（排序靠前 = 步行友好，雨天 / 极端天气都方便出行）。

## 🆕 新增

- `scripts/crawl_amap_metro_walk.py`：高德 `/v3/direction/walking` 拉
  community → 最近地铁站的步行距离 + 步行分钟
  - quota 友好：49 小区 ~ 49 次 API 调用（远低于 5000-30000）
  - quota 用尽或计算失败时自动 fallback 到启发式 (直线 × 1.45 / 80m·min⁻¹)
  - 支持续跑 + 每行立即写盘
- `static/seed/metro_walk.csv`：

  | 列 | 说明 |
  |---|---|
  | community_id | 小区 ID |
  | city_id | 城市 ID |
  | community_name | 小区名 |
  | station_name | 最近地铁站名 |
  | straight_m | 直线距离 (米) |
  | walk_distance_m | 实际步行距离 (米) |
  | walk_minutes | 步行分钟 |
  | source | `AMAP_API` (高德实测) 或 `ESTIMATED` (启发式估算) |

- `LocalMetroWalk` 类型 + `parseMetroWalk` 导入
- `store.getMetroWalks()` / `getMetroWalksByCity()`
- `queries.getMetroWalkRanking({ cityId, topN, maxMinutes })`
- Dashboard "🚶 地铁步行通勤 Top · {city}" 卡片

  - 颜色三档：
    - 🟢 **≤ 5 min** (真正地铁上盖，出门即上车)
    - 🟠 **≤ 10 min** (步行方便，下雨也不会太窘)
    - 🔴 **> 10 min** (需骑车 / 公交接驳)
  - 行内展示：`{min}min / {m}m / 来源：高德|估算`
  - 顶部汇总：城市均值 + 最快 + 总小区数

## ⚙️ 为什么这个数据重要

之前只有 `transit_minutes` (通勤时间，含公交+地铁)，但很多买房用户其实更关心：

> "步行到地铁几分钟？下雨天不撑伞能不能到？"

`metro_walk.csv` 直接答这个 — 是买房决策里**最实际**的"地铁属性"之一。
