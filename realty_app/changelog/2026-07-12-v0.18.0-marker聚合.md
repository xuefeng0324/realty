# v0.18.0 — Marker 聚合（map-2）

> 2026-07-12 · 4h · 🟡 中 · 1km 内多 marker 合并

## 背景

之前 listings 模式直接渲染每套挂牌一个 marker（最多 200/城市）。当用户看深圳全城时，1286 套挂牌挤在一个 marker 层上：
- DOM 节点过多导致渲染卡顿
- 同小区多套挂牌完全重叠，无法看出密度
- 用户需要看的是「这里有多套挂牌」，而不是「这里有 5 个一模一样的小圆点」

本次引入 **网格聚合**（grid-based clustering）：
- 同一网格内的 marker 合并为 1 个 cluster marker
- cluster marker 显示数量（红色气泡）
- 点击 cluster marker → zoom in +1 + 居中到该 cluster 中心
- zoom level 越大（看得越近），网格越小（聚合越少）

## 算法 (src/local/cluster.ts)

```typescript
// zoom 11 (城市级) → cell ≈ 0.04° (≈ 4km)
// zoom 14 (小区级) → cell ≈ 0.005° (≈ 500m)
// zoom 16+        → cell ≈ 0.00125° (≈ 130m) — 几乎不聚合

export function clusterCellDeg(zoom: number): number {
  return 0.04 / Math.pow(2, Math.max(0, zoom - 11));
```

为什么不依赖 uni-app 自带的 marker cluster：
- uni-app 的 `initMarkerCluster` 仅在 App / 小程序支持，H5 不支持
- uni-app H5 用的是高德 JS API，我们直接计算网格更可控、更稳定

## 已知限制

- 聚合算法仅考虑经纬度，不考虑 zoom level 之外的视觉密度（屏幕外 marker 也参与聚合）
- click handler 通过 callout content 区分 cluster vs 单点 (regex `/^(\d+)\s*套$/`)，依赖 UI 文本格式
- 高德 H5 不能完全自定义 marker DOM（无 innerHTML），所以 cluster 数量用 callout 显示而非 marker label
