# v0.3.1 — CI 修复 + 构建健壮性

**日期**: 2026-07-12
**类型**: fix / chore
**版本**: v0.3.1 (补丁版，CI 阻塞修复 + 构建完整性)

## 为什么有这版

v0.3.0 引入 actionlint 后 CI 在 PR #2 / #3 反复失败：

```
Unable to resolve action 'rhysd/actionlint@v1', unable to find version 'v1'
```

根因：`rhysd/actionlint` 仓库本身**不是** GitHub Action（无 `action.yml`），
所以 `uses: rhysd/actionlint@v1` 永远解析不到任何版本。
按官方文档（[usage.md](https://github.com/rhysd/actionlint/blob/main/docs/usage.md)），
正确做法是用 `download-actionlint.bash` 拉二进制。

本批一次性把 CI 阻塞修了 + 顺手做几个工程改进。

## 改了什么

### 1. CI 修复（必修）

`.github/workflows/realty-app-tests.yml`：

- 删除 `uses: rhysd/actionlint@v1`（死引用）
- 改用官方 `download-actionlint.bash` 下载二进制 + `${{ steps.get_actionlint.outputs.executable }}` 执行
- 默认 lint 整个 `.github/workflows/`，含 crawl-* workflow

### 2. CI 加速（crawl workflow）

- `.github/workflows/crawl-daily-wangqian.yml`：`setup-python` 加 `cache-dependency-path: realty_app/scripts/crawl_daily_wangqian.py`
- `.github/workflows/crawl-weekly.yml`：同上指向 `crawl_anjuke.py`

效果：cache key 由依赖文件 hash 决定，依赖不变时秒级命中（之前要重装 requests/beautifulsoup4/lxml）。

### 3. 工程改进（手机端）

- `realty_app/scripts/check.ps1`：在 type-check / test / smoke 之前加 Node/npm 预检。
  缺 Node 时直接给 winget 安装指引，避免后面 npm 报一堆看不懂的错。
- 新增 `realty_app/tests/buildIntegrity.test.ts`：vitest 跑 build 完整性检查。
  - `index.html` 必须有 `<link rel="icon">`、`<div id="app">`、入口 `/src/main.ts`（防 favicon 404 回归）
  - `src/manifest.json` 字段类型合法、`vueVersion === '3'`
  - `static/seed/` 下 5 个 CSV 与 README 远程 CSV 段落声明一致（防改名后文档脱节）
  - `package.json.scripts` 中 `test:coverage` / `type-check` 存在
  - `manifest.versionName` 与 `package.json.version` 大版本对齐
  - CI 必装文件 `tests/e2e/smoke.mjs` 与 `tests/expected.json` 存在

## 设计选择

| 决策 | 选择 | 理由 |
|------|------|------|
| actionlint 版本策略 | 跟随 latest（不锁） | 上游频繁发版（平均每月 1-2 次），锁版本反而落后；锁法见官方 download script 用法 |
| cache-dependency-path 写法 | 单行字符串 | setup-python v5 接受单文件路径，列表写法要求有 requirements.txt，本仓库没有 |
| buildIntegrity 测试位置 | `tests/buildIntegrity.test.ts`（不走 e2e） | 纯文件读取 / JSON 解析，毫秒级；放 e2e 会拖慢 CI |
| Node 预检退出码 | 缺 Node 时 `exit 1`（不继续） | 后续 npm / tsc / vitest 全会失败，提前 fail fast |
| 版本号 | v0.3.1（补丁版） | 不破坏 v0.3.0 已发版的功能，仅修复 CI；不强制触发 App 重打包 |

## 不做什么

- **不**给 actionlint 加缓存（单次 download 约 5MB，几秒完成，不值得）
- **不**为 crawl workflow 写 requirements.txt（依赖少且稳定，inline `pip install` 反而更直观）
- **不**把暗色主题 palette 提取到独立 CSS（这是更大重构，等下次专门迭代）

## 验证

- 本机 Node 20+ 安装后跑 `npm test` 应包含新测试（约 50+ 用例）
- 推送到 main 后 `realty-app-tests.yml` 的 actionlint 步骤应秒过
- 后续 PR 触发 `crawl-*.yml` 时 cache 命中应秒跳过 pip install
