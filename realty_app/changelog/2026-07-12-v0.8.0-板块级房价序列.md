# v0.8.0 — 板块级房价序列（dashboard 区级趋势卡片）

> 日期：2026-07-12
> 关联：OPTIMIZATION_BACKLOG.md F 段

## 背景

上一版（v0.7.0）已经把"未来周边地铁"做进了 listing/community 详情页。
但 dashboard 上只有"全国 70 城指数"和"政府每日网签"两类宏观/网签数据，
**没有"用户在意的具体区"的真实房价走势**——这是地产分析最核心的需求之一。

国家统计局只发布"市辖区整体"指数，**不区分到板块**；
链家/贝壳的成交均价数据是商用的，
我们手头能拿到颗粒度最细的就是 `listings.csv`（1286 条，v0.3.0 + v0.4.0 增量）。

## 决策

**复用 listings.csv**：按 `(city_id, district_name, week_end)` 聚合均值 / 中位数 / 房源数，
生成 `district_trend.csv`（区级周维度价格序列）。
展示到 dashboard。

数据源：
- 链家在售挂牌（每条含 `community_id` → 关联 `communities.csv` → 拿到 `district_name`）
- 80% 真实楼盘的 1226 条 v0.3.0 种子 + 60 条 v0.4.0 真链家
- 共 1286 条 listings → 269 个 (区/周) 组合 → **15 个区 × 27 周**

输出文件：`static/seed/district_trend.csv`（schema 见下）

```
city_id, district_name, week_end, listing_count,
avg_unit_price, median_unit_price, min_unit_price, max_unit_price
```

例：`1,天河区,2026-01-04,6,72359.83,70578.0,67767.0,81537.0`

## 脚本

`scripts/compute_district_trend.py`（标准库，~80 行）：

1. 加载 communities.csv，建 `community_id → district_name` 索引
2. 加载 listings.csv，过滤 `unit_price > 0` + `crawl_date` 有效
3. 按 (城市, 区, 周日) 聚合 avg / median / min / max
4. 写到 district_trend.csv

跑一次只需要几秒。

## 数据质量

- **天河区** (广州)：27 周样本，4 套/周，波动 ±5%（稳定）
- **南山区** (深圳)：9 周样本，**1-3 套/周**（波动 -21.7%，样本太少）
- **福田区** (深圳)：样本稳定在 2-5 套/周

样本量是主要瓶颈——一旦链家/贝壳接入更多真房源，趋势会显著改善。

## 局限

- **没有官方区级指数**——这是 NBS（国家统计局）刻意不发布的颗粒度
- **样本量太小**——每周 1-10 套，统计显著性弱，仅供参考
- **没有户型/楼龄分层**——同样区里 90㎡ 改善 vs 50㎡ 刚需差距大
- **没有成交 vs 挂牌拆分**——挂牌价含议价空间
