# v0.31.0 — new-9 生活便利度 (life convenience)

**日期**: 2026-07-13
**作者**: xuefeng0324
**关键词**: POI 综合打分 / 5 维评分 / 复用现有 poi_seed.csv

---

## 1. 背景

dashboard 已有 30+ 张数据卡，但**没有任何一张卡**把周边 POI 综合起来给一个"小区生活便利度"分数。用户最关心的就是"住这里方便吗"，所以我们用现成的 `poi_seed.csv` (5 类 POI: mall/park/subway/school/hospital) 做加权打分，输出一张可读榜。

## 2. 数据源

- **输入**: `static/seed/poi_seed.csv` (v0.5.0 起已有, 5 类 POI)
- **脚本**: `scripts/compute_life_convenience.py`
- **输出**: `static/seed/life_convenience.csv` (52 行, 3 城全覆盖)

### 评分规则 (满分 100)

| 维度   | ≤500m | ≤800m | ≤1000m | ≤1500m | ≤2000m | ≤3000m | > |
|--------|-------|-------|--------|--------|--------|--------|---|
| 商场   | 25    |       | 20     | 10     |        |        | 0 |
| 公园   | 15    |       | 10     | 5      |        |        | 0 |
| 地铁   |       | 15    | 20     | 5      |        |        | 0 |
| 学校   | 10    |       | 7      | 4      |        |        | 0 |
| 医院   |       |       | 15     |        | 10     | 5      | 0 |

地铁 ≤300m 加 25 分, 总 max = 25+15+25+10+15 = 90, 但 5 维合计最大 100

## 3. 实现

### 3.1 Python (`scripts/compute_life_convenience.py`)

```python
THRESHOLDS = {
    "mall": [(500, 25), (1000, 20), (1500, 10), (99999, 0)],
    "park": [(500, 15), (1000, 10), (1500, 5), (99999, 0)],
    "subway": [(300, 25), (500, 20), (800, 15), (1500, 5), (99999, 0)],
    "school": [(500, 10), (1000, 7), (1500, 4), (99999, 0)],
    "hospital": [(1000, 15), (2000, 10), (3000, 5), (99999, 0)]
}
```

取每个 community 每类 POI 最近距离，套阈值算分，求和。

### 3.2 TS 数据层

- `LocalLifeConvenience` (types.ts)
- `parseLifeConvenience` (importer.ts)
- `getLifeConveniences()` / `getLifeConveniencesByCity()` (store.ts)
- `getLifeConvenienceRank({ cityId, topN, minScore })` (queries.ts)

### 3.3 UI (`dashboard.vue`)

新卡 **🧭 生活便利度 Top 小区**

- 城市均分 + 最高分 (顶部 summary)
- 每行: 小区名 + 区 + 5 维评分 (M/P/S/X/Y) + 总分
- 颜色档: ≥80 绿, 60-79 蓝, <60 灰
- 默认 Top 8 (按综合分降序)

### 3.4 CSS

新增 `.lc-row`, `.lc-dim`, `.lc-score` 系列样式, 与现有深色主题一致。
