# v0.3.0 优化批次-5 — actionlint 接入

**日期**: 2026-07-12
**类型**: CI 自检 / 预防性
**版本**: v0.3.0 (同一版本号下第五批优化)

## 改了什么

在 `.github/workflows/realty-app-tests.yml` 里加一个 `rhysd/actionlint@v1` 步骤，
对整个 `.github/workflows/` 做静态检查：YAML 语法、shell 脚本语法（`run:` 块）、
`uses:` 引用的 action 是否存在、`on:` / `jobs.` / `steps.` 结构是否合法。

### 为什么

仓库根 `.github/workflows/` 下已经积累 3 个 workflow（crawl-daily-wangqian、crawl-weekly、
realty-app-tests），未来还会加。手写 YAML 容易踩这些坑：

| 坑 | actionlint 报错示例 |
|----|---------------------|
| `steps.id` 引用错 | `property "X" is not defined` |
| `uses: actions/checkout@master` | `use the full length commit sha instead of the branch` |
| `run: ...` 里 `set -e` 漏写 | `shellcheck warning SC2086` |
| matrix 引用未声明的变量 | `property "X" is not defined in object` |
| `if:` 表达式语法错 | `expression is not valid` |

### 设计选择

| 决策 | 选择 | 理由 |
|------|------|------|
| 步骤位置 | `setup-node` 之后、`npm ci` 之前 | fail fast；lint 是秒级，依赖安装是分钟级，先 lint 再装依赖省时间 |
| lint 范围 | 默认（整个 `.github/workflows/`） | 覆盖 `crawl-*` workflow，本 workflow 通过不代表其它通过 |
| 参数 | `ignore-version-of-actions: false` | 默认即可；要锁版本时应改 `pin-actions` 步骤而不是这里 |
| 失败处理 | 默认（非零 exit fail job） | lint 警告一般不阻塞，但 actionlint 在 release 间会有破坏性变化，宁严勿松 |

### 不做什么

- **不本地装 actionlint** — Go 二进制 5MB+，不值得给一次性检查装。CI runner 才是 lint 主场
- **不用 pin-github-action** — 锁 SHA 太严，refactor 友好性差；后续若真有 supply chain 顾虑再上
- **不加 commit message 检查** — 那是 `commitlint` 的事，CI 之外不需要

## 验证结果

- YAML 结构手写对齐 2 空格 indent；line 数 72 → 79（加 7 行步骤 + 注释）
- `rhysd/actionlint@v1` 是 Go 官方维护的最新稳定版（v1.x 长期支持）
- 触发后 CI runner 上会自动跑 — 本批 push 是首次跑点
