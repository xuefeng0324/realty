# v0.37.0 5 维小区指标 (listing 列表 + community 详情)

> 2026-07-13

## 🎯 这次改了什么

把分散在 5 个 dashboard 卡片里的指标"内聚"到 listing 列表 + 小区详情 2 个最常看的页面中，让用户**在浏览房源时就能看到**该房源/小区的"生活便利度 / 学区 / 通勤 / 步行地铁 / 规划地铁"5 项关键指标。

## 🆕 新增

### Listing 列表 mini 评分条 (5 维)

每个房源卡片底部新增 5 列迷你评分条：

| 位置 (location_score) | 房屋 (house_quality) | 楼龄 (building_age) | 配套 (amenity) | 性价比 (price_value) |

颜色分档：🟢 ≥75 / 🟠 ≥50 / 🔴 <50

数据来自 `queries.toListingItem` 已经有的 `explain_preview.dimension_scores`（listing 评分系统的 5 维明细）。

### Community 详情 5 维指标卡 (小区维度)

新增 dashboard 风格的 5 格小区指标卡：

| 🧭 生活 | 🎓 学区 | 🚌 通勤 | 🚶 步行地铁 | 🚇 规划地铁 |

数据源：
- 生活：`life_convenience.csv` (life_score100)
- 学区：`school_premium_community.csv` (avg_school_score)
- 通勤：`commute.csv` (transit_minutes) → 换算 `(transit_min - 30) × (50/30)` 反向
- 步行地铁：`metro_walk.csv` (walk_minutes) → 5min=100 / 30min=0
- 规划地铁：`metro_benefit.csv` (benefit_score)

**示例（中海天钻, id=7 京基100）**：
- 生活 81 / 学区 100 / 通勤 100 / 步行 90 / 规划 0

**示例（保利天悦, id=15）**：
- 生活 0 / 学区 0 / 通勤 84 / 步行 99 / 规划 90

颜色分档同 listing 一致。
