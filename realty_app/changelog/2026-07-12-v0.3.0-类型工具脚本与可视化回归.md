# v0.3.0 优化批次-2 (2026-07-12)

**类型工具收尾 + 工程脚本 + 可视化回归**

承接上一批"类型收敛 / a11y / CSS 一致性"之后的进一步工程化。

## 变更摘要

### 1. 类型层：apiPost/apiGet 收紧到 unknown + errorMessage 工具

- `src/api/http.ts`：
  - `ApiError.data`、`buildUrl` params、`apiGet<T>` / `apiPost<T>` 默认泛型：全部从 `any` 改为 `unknown`
  - 新增 `asPayload<T>(data)` 内部辅助函数做 narrow(校验后端返回是 object/string/number/boolean)
  - non-H5 分支 `uni.request.data` 的 cast 显式标注 `string | Record<string, unknown> | undefined`
- `src/utils/errorMessage.ts` 新增 `toErrorMessage(e: unknown)` 工具函数，封装 `e?.message || String(e)` 模式，让 catch 类型可以从 `any` 收紧到 `unknown`。**注**：本次各页面 catch 调用点迁移因 PowerShell UTF-8 兼容问题暂未批量应用，工具已就位待后续单文件迁移。

### 2. a11y 收尾：6 个页面补 role / tabindex / hover-class

把上批未做的页面补齐：
- `pages/listing-detail/listing-detail.vue`：评分解释折叠 row 加 `role="button"` / `tabindex` / `hover-class="row-active"` / `tap-target`
- `pages/school/school.vue`：school-row 同样三件套
- `pages/stats70/stats70.vue`：setKind tag × 2、setBase tab × 2、toggleSortDir、rank-row 全补齐
- `pages/wangqian/wangqian.vue`：link-card(深圳预售入口) + sheet-item 弹层项
- `pages/settings/settings.vue`：toggleAdvanced row + 数据源类型 picker
- `pages/gov-webview/gov-webview.vue`：已是 `<button @click>`，无需改

### 3. 新增工程脚本

- `realty_app/scripts/check.ps1`：一键跑 type-check + unit test + E2E smoke，每步独立 try/catch 隔离失败，退出码 0/1 反映全部通过与否
  - 用法：`powershell -File scripts/check.ps1` / `-SkipSmoke` / `-SmokeUrl "http://127.0.0.1:9001/"`
- `realty_app/scripts/commit.ps1`：参数化 git plumbing commit，绕开 Cursor agent 注入 `--trailer` 与 Git 2.24 不支持的兼容问题
  - 用法：`powershell -File scripts/commit.ps1 -Message "..." -Add "path/to/file" -Push`
  - 支持 `-MessageFile`（长消息从文件读）、`-DryRun`（只打印不执行）

### 4. Vitest coverage 接入

- `package.json` devDependencies 加 `@vitest/coverage-v8@^2.1.9`(与项目 `vitest@^2.1.9` 对齐)
- `vitest.config.ts` 加 coverage 配置（provider: v8），exclude `src/pages`/`src/store`/`src/main.ts`/`.d.ts`
- `package.json` scripts 加 `test:coverage` 一键出报告
- **当前覆盖率**（vitest + 71 tests）：
  - `src/rules/` 评分逻辑 95.86% — 评分核心已扎实覆盖
  - `src/local/` 82.03% — CSV / seed / store / query 都有测试
  - `src/api/` 0% — HTTP 客户端主要靠 pipeline.test 间接覆盖（真实漏洞，待补单测）
  - `src/pages/` 0% — uni-app 组件测试较重，本项目暂不做，靠 E2E 替代

### 5. 可视化回归：tests/e2e/visual-diff.mjs

- 基于 sharp 做像素级 diff，对比 `smoke.png` 与 baseline
- `--update-baseline` 更新 baseline；`--threshold 0.01` 调整容差（默认 1%）
- 缺 sharp 时 fallback 到字节 hash（粗粒度）
- 产出 `tests/e2e/artifacts/diff.png` 高亮差异区域、`visual-diff.json` 报告
- `tests/e2e/artifacts/.gitignore` 调整：`baseline.png` 入版本，其余 png/json 出版本

### 6. 工具/依赖

- `package.json` 加 `sharp`(devDep，仅 visual-diff 用)
- `.gitignore` 加 `realty_app/coverage/`(coverage 产物不入版本)
- `.gitignore` 规则细化：`/.git_commit_msg_*.txt` 专门忽略 commit.ps1 临时 message
