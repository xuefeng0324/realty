# v0.9.0 — 地图找房（高德 JS API + 小区点 + 热力图）

> 日期：2026-07-12
> 关联：OPTIMIZATION_BACKLOG.md G 段

## 背景

上一版（v0.8.0）已经做了 dashboard "区级近 8 周房价趋势"。
但用户进入 app 的核心动作是 **"在地图上找房"**——这是链家/贝壳/安居客最常用的入口。
本仓库的 listings.csv 已经有关联 community_id + communities_geo.csv 有 52 个小区的经纬度，
**数据现成，缺的只是一个 UI 入口**。

调研了 4 个开源项目：
1. `xjkj123/Lianjia`：链家地图 API（v0.4.0 验证 404，已下线）
2. `cjx6668845/lianjia-beike-spider`：21 城爬虫，无地图集成
3. `AnyMarvel/LianjiaSpider`：链家 App 签名破解，无地图
4. **uni-app 官方 `<map>` 组件** + 高德 JS API（H5）：免费、易集成

## 决策

**用 uni-app 内置 `<map>` 组件 + 高德 JS API（H5 平台）**

理由：
- 我们**已经有高德 Web Services key**（`f22d0a9e25abc8512dbdbe37ac3ba139`，v0.4.1+ 在用）
- uni-app H5 平台在 `manifest.json` 配置 `h5.sdkConfigs.maps.amap.key` 后自动加载高德 JS API
- 不需要新装依赖（uni-app 内置）
- App 端走原生高德 SDK（待配置独立 App key）

## 功能

新增 **`/pages/map-view/map-view`** 页面，含：

### 双模式

| 模式 | 渲染 | 适用 |
|------|------|------|
| **热力图** | circles（半径 200-1000m，颜色按挂牌数：红=多/蓝=少） | 找热门板块 |
| **挂牌点** | markers（每套挂牌一个点 + 气泡） | 看具体房源分布 |

### 交互

- 顶部城市切换按钮：深圳 / 广州 / 珠海
- 模式切换按钮
- 点击 marker → 弹出底部 info-card → 跳小区详情
- 自动定位到当前选中城市

### 数据

- **52 个小区**有经纬度（communities_geo.csv）
- **1286 套挂牌**关联到小区
- 热力图按"挂牌数"分级（200-1000m 半径）
- 挂牌点限制 200 个/城市（避免渲染卡顿）

## 局限

- **App 端未配置** — 需要申请独立的高德 Android/iOS App key，并打包自定义基座
- **小程序不支持** — uni-app 微信小程序走腾讯地图坐标系，需要单独适配
- **挂牌点限制 200/城市** — 防止渲染卡顿；如要全量需要分页加载或聚类
- **marker 无聚合** — uni-app `<map>` 组件的 `initMarkerCluster` 仅 App/小程序支持
