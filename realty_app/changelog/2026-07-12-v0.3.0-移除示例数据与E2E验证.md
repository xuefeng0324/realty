# v0.3.0 (2026-07-12)

**移除示例数据 + 端到端 E2E smoke 验证**

## 变更摘要

### 1. 数据源一致性：移除程序化 demo

- 删除 `src/local/demoData.ts`（5823 字节）：原文件程序化生成 1 城市/5 小区/3 学校/30 房源，是「星河湾/翠湖天地/翰林院/云栖谷/锦绣城」等小区名 + 完全随机价格
- 启动默认始终为 `buildSeedSnapshot()` 政府公开种子（广州/深圳/珠海 3 城 × 23 个真实小区 × 1226 套真实房源）
- 设置页 `pages/settings/settings.vue`：
  - 模板删除「切到示例数据」按钮
  - `DataMode` 联合类型由 `"seed" | "demo" | "csv-url" | "http"` 缩为 `"seed" | "csv-url" | "http"`
  - `dataModeLabels` 数组去除「示例数据（内置）」项，索引同步重排
  - `pickDataMode` / `dataModeIndex` 减少 `demo` 分支
  - `downloadNewCsv` 中兜底 `setSnapshot(buildDemoSnapshot())` 改为「当前数据源无需手动下载」提示
  - 删除整个 `resetToDemo()` 函数
- README 中目录树 `local/demoData.ts` 行移除

### 2. 测试：pipeline.test.ts 全部走真数据

- `beforeAll` 从 `setSnapshot(buildDemoSnapshot("TestCity"))` 改为 `setSnapshot(buildSeedSnapshot())`
- 删除 13 行「end-to-end seed snapshot queries」describe 块的复制粘贴副本
- 「end-to-end demo queries」describe 改名为 `end-to-end seed query API (guangzhou)`，移除 `beforeEach(buildDemoSnapshot)`，所有断言改为验证 seed 真数据：
  - `getCities` 期望包含 `广州/深圳/珠海`（替换「仅 TestCity」）
  - `getCommunityPriceTrend({ communityId: 1 })` 改为「长度 > 0 即可」（原断言「星河湾」写死为 demo 数据）
  - `searchSchools({ cityId: 2, q: "实验" })` 改为只断言结构（深圳 seed 校名可能不含「实验」）
- 测试套件仍 71/71 通过

### 3. 新增 Playwright E2E smoke 验证

- `tests/e2e/smoke.mjs`：用 Playwright（chromium 1228，channel 模式）打开 `http://localhost:5174/`，监听 `console` / `pageerror` / `response` / `requestfailed`，截图存盘 + JSON 报告
- 移动端视口（390×844 iPhone 14）+ 中文 UA，确保看到与 HBuilder X 实际渲染一致
- exit code 反映状态：0=pass / 1=有错 / 2=崩溃
- `tests/e2e/make-favicon.mjs`：最小 ICO 生成工具（备用，本次未直接使用）
- 截图与 JSON 产物写到 `tests/e2e/artifacts/`，仓库已 `.gitignore` 忽略 `artifacts/`

### 4. 修复浏览器 favicon 404

- `index.html` 加 `<link rel="icon" href="data:," />`，浏览器不再请求 `/favicon.ico`，console 无 404 红字
- 选择 `data:` 方案而非生成静态 favicon：`static/` 在 uni-app h5 模式不被 Vite 直接服务
