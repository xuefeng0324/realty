# v0.57.0 数据可信度与首页减负

日期：2026-07-18

## 背景

审查发现内置 listings 中多数记录是由公开城市指标派生的分析样本，但旧 `source` 文案容易被理解为住建局逐套成交；远程刷新又只更新 listings，导致榜单与画像继续使用旧派生 CSV。首页默认加载 35 张卡片，视觉回归和 smoke 在 CI 中也不会阻塞错误。

## 主要变更

### 数据可信度

- `listings.csv` 增加 `source_kind`：`REAL / DERIVED / ESTIMATED / UNKNOWN`。
- 1226 条程序派生记录改名为“广州/深圳/珠海公开指标派生样本”；60 条链家挂牌标记为 `REAL`。
- listing 详情页增加可信度 badge；派生样本显示“不代表真实逐套挂牌或成交”的说明。
- Dashboard 顶部显示当前城市真实挂牌、派生样本和其它数据数量。
- 天气卡从“实时天气”改为“天气快照”，展示数据新鲜度；湿度/风力/温度推断从“AQI”改称“扩散条件”。

### 快照一致性

- 新增 `snapshotLoader.ts`，并行加载 5 个必需基础 CSV 和全部可选派生 CSV。
- `dataRefresher.ts` 改为完整快照原子替换，校验 meta 行数和小区关联后才写入 store。
- `publish_csv.py` 增加 `schema_version=2`、`snapshot_sha256`、逐文件行数和文件数。
- 新增 `rebuild_listing_derivatives.py`，按依赖顺序运行 16 个派生脚本。
- weekly workflow 在发布前重建派生数据，并提交整个 `static/seed/*.csv`。

### 数据模式

- 移除未真正接入页面查询层的 HTTP 模式，保留“内置完整快照 / 自定义 CSV 快照”。
- 修复空 storage 被误显示为 HTTP 模式的问题。
- 自定义 CSV 模式在启动时自动恢复；快照替换后通知 dashboard、房源、学校和地图页重新加载。

### 排版与回归

- Dashboard 默认 tab 从“全部”改为“概览”，保留数据覆盖、区对比、网签、通勤、综合评分、学区和 LPR 等关键卡。
- hero 轮播改用 uni-app `swiper`，移除固定 `scrollLeft / 270` 计算。
- 视觉 baseline 从旧的 390×1743 更新到当前概览布局。
- smoke 和 visual diff 失败重新阻塞 CI；静态服务目录修正为 `dist/build/h5`。

### 其它修复

- 70 城日期改为数值年月日排序，修复 10～12 月趋势被排到 2～9 月前的问题。
- package、package-lock、manifest、设置页版本统一到 0.57.0。
- 高德 Web Service 爬虫不再内置默认 Key，要求使用 `AMAP_KEY` 或 `--key`；H5 JS 地图 Key 保留在 manifest。

- `node tests/e2e/smoke.mjs`：pass，无 console/page/network error
- `node tests/e2e/visual-diff.mjs --threshold 0.02`：更新 baseline 后 byte-identical
