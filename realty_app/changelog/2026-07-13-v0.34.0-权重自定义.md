## 1. 背景

v0.33.0 综合评分固定权重 (生活50%/学区30%/通勤20%)，但**不同用户的需求差异大**：
- 学龄家庭重视学区 (小孩上学)
- 上班族重视通勤 (时间就是金钱)
- 老人重视生活 (医院/菜市场)
- 投资客重视地段 (综合均衡)

本版本让用户能根据自身需求调权重，看到不同偏好下的 Top 小区变化。

## 2. UI

dashboard 「🏅 小区综合评分 Top 小区」卡新增:

### 2.1 4 个预设 chip
- **⚖️ 均衡** (50/30/20) - 默认
- **🎓 学区** (20/60/20) - 学龄家庭
- **🚇 通勤** (20/20/60) - 上班族
- **🧭 生活** (70/20/10) - 居家养老

### 2.2 3 个 slider
- 生活 (蓝) 0-100, step 5
- 学区 (绿) 0-100, step 5
- 通勤 (黄) 0-100, step 5

### 2.3 当前权重显示
```
当前权重：生活 50% · 学区 30% · 通勤 20%
```
若总和 ≠ 100，显示警告并自动归一化。

## 3. 算法

```typescript
const wSum = weights.life + weights.school + weights.commute || 100;
const wf = {
  life: weights.life / wSum,
  school: weights.school / wSum,
  commute: weights.commute / wSum
};
totalScore = round((life*wf.life + school*wf.school + commute*wf.commute) * 10) / 10;
```

- 总和归一化 (避免用户拖到 50+30+10=90)
- 实时重算 + 重排 rank_city
- 用 `setTimeout` 简单 debounce (slider 拖动时不会刷爆)
