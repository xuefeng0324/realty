**日期**: 2026-07-12
**基线**: v0.3.1 (commit 2870b63)
**变更范围**: 性能 / 健壮性 / 数据质量

---

## 变更目标

seed CSV 中的 `listings.csv` **过去** 是 `seed_real_data.py` 用 random 派生的 **fake 数据**（品牌色块、随机价格），导致：
- 用户看到的深圳福田区 listing 数量、价格结构都跟真实市场不符
- 评测/对比时无法用真实购房者的角度验证 UI
- 不利于发现产品逻辑 bug（如：地区均价排序、地铁距离提示、楼龄分布等）

v0.4.0 把 60 条**真实**挂牌数据混入 seed，原 fake 数据保留（作回归保护 + 现阶段数据规模过小）。

## 关键决策

### 为什么不用 xjkj123/Lianjia 的 ajax API
README 说"无限制调用"，但实测（2026-07-12）`https://ajax.lianjia.com/map/search/ershoufang/` 返回 404，**API 已下线**。改成爬 HTML 列表页：单页 30 条，奇偶页交替触发风控（所以 `--pages 5` 时只稳定拿到 60 条）。

### 为什么纯标准库
仓库 `scripts/` 现有依赖只有 stdlib 的 urllib；本机 pip 受 `distutils-precedence.pth` 损坏影响装不了 bs4/lxml，**纯 stdlib 是最稳的选择**。

### 为什么保留 fake 数据而不是替换
- 单次抓到 60 条真数据替换 1226 条 fake 会让 UI 显得"内容稀缺"
- 现有 114 个单测里 `pipeline.test.ts` 假设**"深圳 latests weekEnd 有 ≥10 个 community 的 ranking"**，需要稳定的数据保证
- 后续提升抓取量（CI runner 全新 IP） + 多城市后，再用真数据替换 fake
- 当前 commit 后 1286 行 = 1226 fake (回归保障) + 60 real (验证流通)

## 已知限制

1. **链家 IP 风控**：单个 IP / 高频请求会被 captcha 弹回；CI runner 全新 IP 会好，但 GitHub Actions 偶尔也会被整体 IP 段拉黑。
2. **community district 缺失**：链家列表页不带 district，加 district 必须抓详情页，留给后续 `enrich_communities.py` 任务。
3. **地铁距离 / 电梯**：列表页 HTML 不显示，需详情页（to be done）。
