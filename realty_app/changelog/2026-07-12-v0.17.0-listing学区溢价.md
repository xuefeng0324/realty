# v0.17.0 — Listing 学区溢价榜（trend-6）

> 2026-07-12 · 1d · 🟡 中 · 板块溢价/学区评分联动

## 背景

之前 v0.11.0「学区溢价榜」是 **区级别**（district 维度），用户能看到「南山区学区好、溢价 +25%」但不知道 *具体哪些房源*。这次新增 **listing 级别** 的学区评分 + 板块溢价，**直接告诉用户哪些房源是学区房 + 高溢价**。

## 数据流

```
listings.csv (1286) + communities.csv (52, district_name) + schools.csv (58, district_name) + school_indicators.csv (58, latest_level_score_raw)
  → 按 (city_id, district_name) 聚合 schools,得到 school_score (avg latest_level_score_raw)
  → 给每个 listing 打 school_score (来自其 community.district_name)
  → 给每个 listing 打 school_count (该区学校数)
  → 算 premium_ratio_est: (区中位单价 / 全市中位单价 - 1) * 100
  → 输出 listing_school_premium.csv (1286 行)
```

> 注：listings.csv 的 school_ids_json 是 fake data (大多 `[]`)，所以本次策略是 **通过 (city, district) 关联**：
> listing → community → district_name → 同区学校聚合评分。这避免了直接依赖空的 school_ids_json。

## 已知限制

- listings.csv 的 school_ids_json 是 fake data (大多 `[]`)。本次用 (city, district) 间接关联，效果是 *该区所有 listing 共享同一学区评分*，并不是 *单 listing 的精确学区*。
- premium_ratio_est 用 (区中位 / 全市中位 - 1) 近似，未考虑小区品质、楼龄等因素。
