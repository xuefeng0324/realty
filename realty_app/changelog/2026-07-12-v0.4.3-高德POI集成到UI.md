# v0.4.3 — 把高德 POI 集成到 listing-detail 和 community 页面（Option B）

**日期**: 2026-07-12
**基线**: v0.4.2 (commit 1c21c65)
**变更范围**: 数据层（types/importer/store/queries） / 两个 vue 页 / 单测 / E2E

---

## 变更目标

v0.4.1 已抓 657 行 POI 数据进 seed，但前端从未使用。B 阶段把数据真正显示到 UI：

| 页面 | 卡片 |
|------|------|
| `pages/listing-detail/listing-detail.vue` | "周边配套" 新卡片：5 类 POI，最近 3 个 + 距离 |
| `pages/community/community.vue` | 同上卡片放在"质量分布"与"优点 Top"之间 |

---

## 数据层接入（5 个文件）

### 1. `src/local/types.ts`
- 加 `LocalPoi` 类型（9 字段：community_id, category, rank, name, type, distance, lat, lng, address）
- `DataSnapshot` 加 `pois: LocalPoi[]` 字段

### 2. `src/local/importer.ts`
- `SnapshotInputs` 加可选 `poisCSV`
- `importSnapshot` 解析 `poi_seed.csv` → `LocalPoi[]`，校验 5 类 category & 必填数值字段
- 可选：缺 `poisCSV` 时装 `[]`（向后兼容 settings.vue 的 CSV URL 模式）

### 3. `src/local/store.ts`
- 加 `getPoisByCommunity(communityId)` 按 (category, rank) 排序
- 加 `getTopPoisByCategory(communityId, category, limit=3)`

### 4. `src/local/queries.ts`
- 加类型 `PoiCategory`, `PoiItem`
- 加 `getCommunityPois({communityId}) → {items}`

### 5. `src/local/dataRefresher.ts`
- 修：`mergeListingsIntoSnapshot` 保留 `existing.pois`，避免远端 refresh 时丢失本地 POI

### 6. `src/local/seedSnapshot.ts`
- import `poi_seed.csv?raw` 并塞进 `importSnapshot(poisCSV)`

### 7. `src/pages/settings/settings.vue`
- csv-url 模式拉远端时也顺带 fetch `poi_seed.csv`（失败用空字符串兜底）

### `community.vue`

- onMounted 与其他调用并行追加

## 不在 v0.4.3 / 已知缺口

1. **24 号中核集团宿舍无 subway POI**：因为该区域远离地铁站（高德"around" 默认 1km 半径；该地点 1km 内可能确实没站）。future 改进：扩半径到 2km。
2. **POI "around" 1km 是硬编码**：v0.4.1 抓数时未暴露 radius 参数。
3. **不足**区下方没显示该小区 POI 总数（如 "5 类 POI 共 13 个"）：UI 简洁已足够。
