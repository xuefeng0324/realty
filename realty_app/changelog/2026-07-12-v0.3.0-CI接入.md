## 改了什么

### 1. 新增 `.github/workflows/realty-app-tests.yml`

GitHub Actions workflow，每次 push/PR 到 main、工作日 09:00 北京时间
排程、手动 `workflow_dispatch` 都会自动跑 type-check + 单测 + coverage。

**核心设计**：

| 项 | 选择 | 理由 |
|---|---|---|
| runner | `ubuntu-latest` | 比 windows-latest 便宜 2 倍，无 PowerShell 依赖 |
| Node 版本 | `20` LTS | 与本机开发版本对齐；CI 跑 `npm ci` |
| 触发器 | push/PR + cron + manual | push 守 PR/CI；cron 守定时回归；manual 守重跑 |
| 缓存 | `actions/setup-node@v4` + `cache-dependency-path` | 复用 `package-lock.json`，二次构建省 30-60s |
| 并发 | `cancel-in-progress: true` | 新 push 进来自动取消旧 run，省 quota |
| timeout | `15 min` | 防 hang |

**步骤**（`working-directory: realty_app`）：

```
1. checkout
2. setup-node v4 (Node 20, npm cache)
3. npm ci
4. npm run type-check
5. npm run test:coverage
6. upload coverage artifact (14 days retention)
7. upload E2E artifacts (7 days retention)
8. 汇总（always 跑，无论成败）
```

## 为什么 E2E smoke 暂不入 CI

`tests/e2e/smoke.mjs` 用 Playwright + Chromium，Linux runner 上：

| 步骤 | 耗时 | 风险 |
|------|------|------|
| `npx playwright install chromium` | 1-2 min | 安装失败 / 镜像抽风 |
| `npm run dev:h5` 启 dev server | 10-30s | 端口占用 / 编译错误 |
| 跑 smoke.mjs | 10-20s | timeout |
| **总计** | **2-3 min / 次** | **失败率高** |

权衡：CI 主要价值是"代码 push 进 main 前一定 type-check + test 通过"。smoke 留给
本机 `scripts/check.ps1 -SkipSmoke:$false` 验证，或单开 workflow（Linux runner + playwright
缓存，下次再做）。

## 验证结果

- workflow 文件 YAML 格式手写无 lint 工具校验（仓库没 yamllint / actionlint），靠仔细对齐 indent
- 触发条件 `paths:` 限定 `realty_app/**` + workflow 文件本身，避免无关 PR 触发浪费 quota
- 本机不需要跑 — CI runner 上才是真验证环境；下次 push 即触发
