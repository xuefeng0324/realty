# v0.4.2 — 接入链家 xiaoqu 真小区源 + 关联链家 listings（realty_app）

**日期**: 2026-07-12
**基线**: v0.4.1 (commit 79187a0)
**变更范围**: 数据来源 / 关联 / 单测 / E2E

---

## 变更目标

v0.4.0 接入链家 60 条真 listings 后，遇到 2 个数据缺口：

1. **链家 listings 的 `community_id` 全部 = 0** — listing 详情页"小区详情"卡片无内容。
2. **链家 xiaoqu 列表页可达但从未被使用** — v0.3.1 seed communities 只有 23 个固定小区，远小于真实城市小区量（深圳≥5000 个）；导致 listings/详情页 join 不到小区。

v0.4.2 实施用户 backlog 中的 **Option C（先 enrich 现有 listings）**，为 4 步流水线：
1. 抓链家 `https://sz.lianjia.com/xiaoqu/pg1/` 真小区列表
2. 写入 `communities.csv` 末尾 (id 24+)
3. 重跑 geo + poi 让新小区也获得经纬度和 POI
4. 把 `community_id=0` 的链家 listings 按城市轮询关联到深圳小区

## 变更明细

### 新增脚本

1. **`scripts/enrich_lianjia_xq.py`** — 链家 xiaoqu 列表页抓取器
   - `fetch --city 深圳 --pages 1`：抓 pg1，约 30 个小区
   - `write --city 深圳`：追加到 `communities.csv`，自动接续 ID（24+）
   - 解析 district / bizcircle （从 `class="positionInfo"` 块）
   - **fallback 处理 `title="福田区小区"` → "福田区"**（去掉"小区"两字）

2. **`scripts/enrich_lianjia_listings.py`** — 链家 listings 的 community_id 关联器
   - `assign`：把 `source="链家在售"` 且 `community_id=0` 的 listings 按城市轮询分配到 cities 现有小区
   - 写回前自动备份 `listings.csv.bak`
   - 关联方式：round-robin，**不**伪造真实对应关系（链家无 API 暴露 listings↔xq_id 映射）
