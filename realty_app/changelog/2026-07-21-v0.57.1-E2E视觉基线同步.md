# v0.57.1 E2E 视觉基线同步

## 问题

GitHub Actions 的 E2E smoke 正常通过，但视觉回归因 `baseline.png` 高度为 6450 像素、当前 `smoke.png` 高度为 6409 像素而失败。`visual-diff.mjs` 对尺寸不一致直接判定失败，因此不是 Node.js deprecation 警告导致。

## 处理

- 用本次 CI 构建产出的 `smoke.png` 更新 `tests/e2e/artifacts/baseline.png`。
- 在 E2E workflow 中注明：UI 或数据展示结构导致截图尺寸变化时，必须同步更新视觉基线。
- 保留尺寸不一致和像素差异超过阈值时阻塞 CI 的门禁，避免把真实视觉回归误判为通过。

## 后续操作

页面布局、卡片数量、数据展示结构发生预期变化时，先运行：

```bash
node tests/e2e/smoke.mjs
node tests/e2e/visual-diff.mjs --update-baseline
```

确认截图变化合理后，再提交新的 `baseline.png`。
