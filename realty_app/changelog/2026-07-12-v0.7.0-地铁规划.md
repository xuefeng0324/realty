# v0.7.0 — 地铁规划 + UI 集成

**日期**：2026-07-12
**类型**：数据 / UI
**前置**：v0.6.0

## 背景

之前只显示"已有地铁"（poi_seed.csv subway 类），没有"未来地铁"信号——
但实际上 **地铁规划** 对买房决策影响巨大（站点 500m 内 = 房价 +5-15%）。

## 数据源

- 深圳五期（2023-2028）：深圳市政府在线（sz.gov.cn）+ 本地宝 + 南都报道（环评二次公示）
- 广州三期调整（2020）：广州市政府门户网站 + 城市交通网 + 澎湃新闻
- 珠海规划：广东本地宝 + 珠海交通

## 设计决策

- **仅当现有最近地铁 ≥ 1km 时显示** — 已有地铁的小区不需要看规划，避免冗余
- **按区粗粒度匹配** — 没有精确站点坐标的小区，按 district 匹配线路 districts 列表
- **手填为主** — 站点级经纬度缺失（公开数据没规范），靠线路名 + 区做匹配
- **不爬高德 POI 二次验证** — 规划线路不存在于高德 POI（建成才有）
