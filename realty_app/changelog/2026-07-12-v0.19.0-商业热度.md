# v0.19.0 — 周边商业配套密度 (new-2)

> 2026-07-12 · 4h · 🟡 中 · 餐饮/银行/便利店 POI + 小区商业热度评分

## 背景

之前 v0.13.0 POI overlay 只覆盖 5 类：🚇地铁 / 🏫学校 / 🏥医院 / 🛍商场 / 🌳公园。
"商场" 是综合商业体，但对 **日常生活便利度**（吃饭 / 取钱 / 买水）帮助有限。

本次新增 3 类商业 POI：
- 🍴 餐饮 (餐厅 / 咖啡 / 奶茶 / 火锅 / 烧烤 / ...)
- 🏦 银行 (ATM / 工行 / 建行 / 招行 / ...)
- 🏪 便利店 (7-11 / 全家 / 罗森 / 美宜佳 / ...)

并基于这 3 类给每个小区算「**商业热度评分**」0..100。

## 数据流

```
高德 /v3/place/around (keywords=餐饮服务/银行/便利店, radius=1000-1500m)
  → scripts/crawl_commercial_poi.py fetch --key <key>
  → 49 小区 × 3 类 × 最多 3 个 = 416 行 POI
  → static/seed/poi_commercial.csv

poi_commercial.csv + communities.csv
  → scripts/compute_commercial_density.py
  → 给每个 community 算商业热度评分
  → static/seed/community_commercial.csv (52 行)
```

## 评分模型 (满分 100)

| 类别 | 0 个 | 1 个 | 2 个 | 3 个 | 满分 |
|------|------|------|------|------|------|
| 餐饮 | 0 | 20 | 35 | 50 | 50 |
| 银行 | 0 | 12 | 22 | 30 | 30 |
| 便利店 | 0 | 8 | 14 | 20 | 20 |

**距离权重** (取每类最近 POI):
- ≤ 300m: × 1.0 (full score)
- 300-800m: × 0.7
- 800-1500m: × 0.4
- > 1500m: × 0.1

```
score = Σ (count_score[category] × dist_weight[nearest_in_category])
```

## 新增文件

### Python 脚本

- `scripts/crawl_commercial_poi.py` — 高德 POI 周边搜索 (3 类: 餐饮/银行/便利店)
  - 配额消耗: 49 小区 × 3 类 = 147 次 (5000-30000 配额内 ✅)
  - 输出: `static/seed/poi_commercial.csv` (416 行)
- `scripts/compute_commercial_density.py` — 计算每个小区商业热度评分
  - 输出: `static/seed/community_commercial.csv` (52 行, 49 有 score > 0)
  - 统计: 94% (49/52) 小区有 score > 0, 平均分 80.4

### CSV

- `static/seed/poi_commercial.csv` — 416 行商业 POI
- `static/seed/community_commercial.csv` — 52 行商业热度
