# v0.10.0 — 网签热度榜（dashboard 新卡片）

> 日期：2026-07-12
> 关联：OPTIMIZATION_BACKLOG.md trend-9

## 背景

上一版（v0.9.0）做了"地图找房"。dashboard 上现在有：
- 全国 70 城指数（宏观）
- 政府每日网签（实时 30 天）
- 区/板块对比（基于挂牌）
- 区级近 8 周房价趋势（基于挂牌）
- 地图（52 个小区）

**还缺一个**：基于**真实成交**的区级活跃度榜。
- daily_wangqian.csv 已经有 **264 条 granularity=district 数据**
- 数据源：深圳市/广州市住建局公示的每日网签（新房/二手）
- 颗粒度：区 × 类别 × 日 → 可聚合到区 × 类别 × 周

## 决策

**复用 daily_wangqian.csv**，按 (city, district, category, week_end) 聚合 units/area_sqm。
展示到 dashboard "近 4 周二手/新房网签热度榜" 卡片。

- **广州市住建局**只公示新房（无二手）→ 自动 fallback 显示新房榜
- **深圳市住建局**同时公示新房+二手 → 优先显示二手榜（更反映市场需求）

## 数据量

| 维度 | 数量 |
| --- | --- |
| 源数据 district 行 | 264 |
| 输出 wangqian_district_weekly.csv 行 | 66 |
| 覆盖城市 | 2 (广州 + 深圳) |
| 覆盖区 | 22 |
| 覆盖周 | 2 (2026-06-28, 2026-07-05) |
| 覆盖 category | 3 (新房 / 二手 / 其他) |

## 脚本

`scripts/build_wangqian_heatmap.py`（标准库，~80 行）：
1. 加载 daily_wangqian.csv (BOM-safe 用 utf-8-sig)
2. 过滤 granularity=district
3. 按 (city, district, category, week_end) 聚合 total_units + total_area_sqm
4. 计算 avg_daily_units, avg_daily_area_sqm
5. 写 wangqian_district_weekly.csv

## 数据层

- `types.ts`：新增 `LocalWangqianDistrictWeekly` 接口 + `DataSnapshot.wangqianDistrictWeekly`
- `importer.ts`：解析 wangqian_district_weekly.csv（含 category 归一化）
- `seedSnapshot.ts`：默认导入

## 关键发现：广州没二手数据

调研时发现**广州市住建局每日网签公示只发布新房，不发布二手**（只有深圳有二手网签）。
代码做了 `preferredCat` 兜底：广州自动用新房榜，深圳默认二手榜。
